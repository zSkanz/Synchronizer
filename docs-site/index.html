State Sync
Made Simple

Server-authoritative data channels with automatic client replication, batched networking, and event-driven updates for Roblox.

⚡

Batched Networking

All mutations are queued and sent as a single payload per frame. Zero wasted bandwidth.

🔒

Server-Authoritative

Data lives on the server. Clients receive read-only caches with rate limiting and validation.

📡

Event-Driven

Subscribe to changes at any path. Supports nested paths, arrays, and dictionaries.

🧹

Zero Memory Leaks

Automatic cleanup on destroy. PlayerRemoving cleanup for rate limit caches.

🔗

Fluent API

Chain mutations together. channel:Set():Set():Increase() — all batched.

🛡️

Strict Typing

Full Luau type annotations for autocomplete and static analysis.

Getting Started

Installation (Roblox Studio)

  1. Create a ModuleScript named Synchronizer inside ReplicatedStorage/Packages
  2. Create a child ModuleScript named Channel inside it
  3. Copy the contents of src/Synchronizer.luau and src/Channel.luau respectively
ReplicatedStorage
└── Packages
    ├── Synchronizer         ← Synchronizer.luau
    │   └── Channel          ← Channel.luau
    └── Signal               ← (dependency)

Via Rojo / Git

git clone https://github.com/zSkanz/Synchronizer.git

Dependencies

Synchronizer requires a Signal module as a sibling inside Packages. We recommend sleitnick/signal.

Basic Usage

Server

local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Players = game:GetService("Players")
local Synchronizer = require(ReplicatedStorage.Packages.Synchronizer)

Players.PlayerAdded:Connect(function(player)
    -- Create a channel with initial data
    local channel = Synchronizer:Create(player.UserId, {
        coins = 0;
        level = 1;
        inventory = {};
    })

    -- Register the player as a listener
    channel:AddListener(player)

    -- Mutate data — clients update automatically
    channel:Set("coins", 100)
    channel:Increase("coins", 50)
    channel:InsertOnArray("inventory", "Sword")
end)

Players.PlayerRemoving:Connect(function(player)
    Synchronizer:Destroy(player.UserId)
end)

Client

local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Synchronizer = require(ReplicatedStorage.Packages.Synchronizer)

local channel = Synchronizer:Wait(game.Players.LocalPlayer.UserId)

-- Listen for changes (forceCall = true fires immediately with current value)
channel:OnChanged("coins", function(newValue, oldValue)
    print("Coins:", oldValue, "→", newValue)
end, true)

channel:OnArrayInserted("inventory", function(item, index)
    print("New item:", item, "at index", index)
end)

-- Read values directly
local coins = channel:Get("coins")
local allData = channel:GetTable()

How It Works

Server Client ┌─────────────────────┐ ┌─────────────────────┐ │ Synchronizer │ │ Synchronizer │ │ ├── Channel A │ FireClient │ ├── Channel A │ │ │ ├── Set() ────┼──(batched/frame)─┼──│ ├── CacheTable │ │ │ ├── Queue ────┼─────────────────►│ │ ├── OnChanged │ │ │ └── Listeners │ │ │ └── Signals │ │ └── Channel B │ │ └── Channel B │ └─────────────────────┘ └─────────────────────┘
  1. Server mutates data via Set, Increase, InsertOnArray, etc.
  2. Mutations are queued and deduplicated within the frame.
  3. On RunService.Stepped, all queued actions are batched and sent to listeners.
  4. Client receives the batch, updates its cache, and fires local signals.

Advanced Usage

Path System

Dot-separated paths navigate nested data structures:

channel:Set("stats.health", 80)
channel:Increase("stats.mana", 10)
channel:OnChanged("stats.health", function(newHP, oldHP)
    print("HP:", oldHP, "→", newHP)
end)

Chaining Mutations

All mutation methods return self for chaining. All three happen in memory immediately but are sent as a single network payload:

channel
    :Set("coins", 500)
    :Set("level", 10)
    :Increase("coins", 100)

ClearSignal — Dynamic Cleanup

Clean up all signals matching a substring when a dynamic entity is removed:

-- Destroy every signal whose path contains "hero_abc"
channel:ClearSignal("hero_abc")

WaitAndCall — Non-Yielding

Unlike Wait, this uses a callback and does not yield the thread:

Synchronizer:WaitAndCall(player.UserId, function(channel)
    channel:OnChanged("coins", function(newValue)
        updateCoinUI(newValue)
    end, true)
end)
print("This runs without waiting!")

Configuration

local Settings = {
    MAX_REQUESTS_PER_SECOND = 10;  -- Rate limit for client data requests
    DESTROY_CLEANUP_DELAY = 5;     -- Seconds before remote cleanup after destroy
}

Security

  • Listener Validation — Clients can only request data from channels they are registered listeners of.
  • Rate Limiting — Clients exceeding MAX_REQUESTS_PER_SECOND are blocked with a warning.
  • Data Validation — Incoming client event data is type-checked before processing.
  • Cache Isolation — Each client has its own cache table, independent of the server.

Synchronizer API

Synchronizer:Create(index, referenceTable?) Server
Creates a new channel with the given index and optional initial data table. If a channel already exists at that index, returns the existing one.
Param index any — Unique identifier for the channel
Param referenceTable {[any]: any}? — Initial data (defaults to empty table)
Returns Channel
local channel = Synchronizer:Create(player.UserId, {
    coins = 0;
    level = 1;
})
Synchronizer:Destroy(index) Server
Destroys a channel and cleans up all associated resources including signals, connections, and listeners.
Param index any — Index of the channel to destroy
Synchronizer:Destroy(player.UserId)
Synchronizer:Get(index) Shared
Returns the channel with the given index, or nil if it doesn't exist.
Param index any — Index of the channel
Returns Channel?
Synchronizer:GetAllChannels() Shared
Returns a dictionary of all active channels.
Returns {[any]: Channel}
Synchronizer:GetTableFromChannel(index) Shared
Returns the raw data table from a channel, or nil if it doesn't exist.
Param index any — Index of the channel
Returns {[any]: any}?
Synchronizer:Wait(index) Shared Yields
Yields the current thread until a channel with the given index exists. Returns immediately if it already exists.
Param index any — Index of the channel to wait for
Returns Channel
Synchronizer:WaitAndCall(index, callback) Shared
Calls the callback when a channel with the given index exists. Unlike Wait, this does not yield the current thread.
Param index any — Index of the channel
Param callback (Channel) → () — Called when channel is ready

Channel API

Channels are created through Synchronizer:Create() and should not be instantiated directly. All mutation methods are server-only and return self for chaining.

Mutations

Channel:Set(path, value) Server Chainable
Sets the value at the given path. Existing values are overwritten.
Param path string — Dot-separated path
Param value any — New value
Returns Channel
channel:Set("coins", 100)
channel:Set("stats.health", 80)
Channel:Increase(path, value) Server Chainable
Increments a numeric value at the given path.
Param path string — Dot-separated path to a numeric value
Param value number — Amount to add (can be negative)
Returns Channel
channel:Increase("coins", 50)
channel:Increase("stats.mana", -10)
Channel:InsertOnArray(path, value, targetIndex?) Server
Inserts a value into an array at the given path. Optionally specify the position.
Param path string — Dot-separated path to the array
Param value any — Value to insert
Param targetIndex number? — Position to insert at (default: end)
Returns number? — The index the value was inserted at
Channel:RemoveFromArray(path, index) Server
Removes a value from an array at the given index.
Param path string — Dot-separated path to the array
Param index number — Index to remove
Returns any? — The removed value
Channel:InsertOnDictionary(path, index, value) Server
Inserts a key-value pair into a dictionary at the given path.
Param path string — Dot-separated path to the dictionary
Param index any — Key to insert
Param value any — Value to insert
Channel:RemoveFromDictionary(path, index) Server
Removes a key from a dictionary at the given path.
Param path string — Dot-separated path to the dictionary
Param index any — Key to remove
Returns any? — The removed value

Listeners

Adds a player as a listener. They will receive all future updates from this channel.
Param listener Player
Returns Channel

Removes a player from the listener list. They will no longer receive updates.

Returns Channel

Getters

Returns the value at the given path, or the full data table if no path is provided.

Returns any

Returns Channel
